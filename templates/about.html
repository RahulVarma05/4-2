About this project

AI-powered motion transfer
from a static portrait

This project implements the First Order Motion Model (FOMM) — a landmark neural network architecture that can animate any object in a source image using only the motion signal from a driving video, without requiring any 3D modelling, rigging, or manual annotation.

📄

Research paper: First Order Motion Model for Image Animation

Siarohin, A., Lathuilière, S., Tulyakov, S., Ricci, E., & Sebe, N. — NeurIPS 2019

Read the paper →
The core idea
What problem does this solve?

Traditional face animation required 3D face models, manual rigging, and expert artists. FOMM solves this with a single neural network trained end-to-end on video data alone.

🖼️

The old problem

Animating a static portrait traditionally required a 3D face scan, bone rigging by an artist, per-expression blend shapes, and hours of manual work per subject. It was inaccessible to most people.

✨

The FOMM solution

FOMM learns to separate appearance (who the person looks like) from motion (how they move) using only raw video pairs. At inference time, motion from any driving video can be applied to any source image.

🔑

Key insight

The model uses unsupervised keypoints — it discovers which parts of a face are important for motion without being told. These keypoints capture facial structure without requiring face-specific priors.

🎯

Why relative motion?

Instead of transferring absolute keypoint positions (which would distort faces of different sizes), FOMM transfers the delta — the change in position from the first frame. This preserves identity while transferring motion.

Pipeline architecture
How the model works, step by step

The pipeline has six distinct stages. Each stage is implemented as a separate module in our clean codebase.

1

Input loading and preprocessing

The source image and each frame of the driving video are loaded and resized to 256×256 pixels. This is the required input resolution for the VoxCeleb-trained checkpoint. Both are normalised to float32 arrays with values in the range 0 to 1.

pipeline/loader.py
2

Keypoint detection

The KPDetector neural network runs on the source image once, and on each driving frame. It outputs 10 keypoints per image — each with an (x, y) position and a 2×2 Jacobian matrix describing local rotation and scale. Keypoints are detected in normalised [-1, 1] coordinates.

pipeline/keypoints.py
3

Temporal smoothing

Raw keypoint positions fluctuate slightly frame to frame due to detector noise. An Exponential Moving Average with alpha=0.60 is applied to smooth keypoint trajectories. This eliminates visible jitter in the output video without introducing motion lag.

pipeline/keypoints.py → temporal_smooth()
4

Relative motion normalisation

The absolute position of driving keypoints is not used directly. Instead, the delta (change) from the first driving frame's keypoints is computed and added to the source image's keypoint positions. This transfers motion without transferring identity — the generated face retains the source appearance.

pipeline/keypoints.py → normalize_kp()
5

Dense motion field and frame generation

The OcclusionAwareGenerator takes the source image, source keypoints, and normalised driving keypoints. It predicts a dense optical flow field (deformation map) and warps the source image features accordingly. An occlusion map is also produced, indicating which regions the generator is uncertain about. The decoder then synthesises the final output frame.

pipeline/motion_transfer.py → make_animation()
6

Compositing and output (motion_transfer mode)

In motion_transfer mode, the generated face is aligned to the driving frame using a SimilarityTransform. A combined mask is built from the ConvexHull of keypoints (shrink=0.85) and the upsampled occlusion map (threshold=0.15). Poisson blending via cv2.seamlessClone (NORMAL_CLONE) seamlessly composites the generated face onto the driving frame, automatically matching lighting and colour at the boundary.

pipeline/masking.py + pipeline/blending.py
Our improvements
What we fixed and added

Six specific bugs in the original demo code were identified and fixed. Three quality improvements were added on top.

Hull mask shrink: 0.60 → 0.85

Old value only covered 60% of the face, creating seam lines across cheeks and forehead. Fixed to 85% for full coverage.

Occlusion threshold: 0.40 → 0.15

Old threshold discarded most valid face pixels leaving a patchy mask. Lowered to 0.15 to recover all confident pixels.

MIXED_CLONE → NORMAL_CLONE

MIXED_CLONE caused driving video skin to bleed back through the face. NORMAL_CLONE takes gradients from source only.

Stable alignment anchor

Old code re-detected keypoints on generated frames causing cascading alignment drift. Now uses stable source keypoints as anchor.

Removed post-Poisson fade

Old code applied alpha blending after Poisson, cancelling the seamless clone and creating a visible halo. Removed entirely.

Temporal smoothing: 0.75 → 0.60

Stronger EMA (40% from previous frame vs 25%) eliminates frame-to-frame jitter without introducing motion lag.

find_best_frame

Starts animation from the driving frame most similar in pose to the source image. Eliminates the frame-0 snap artifact.

Color histogram matching

Per-channel mean+std correction applied before blending. Fixes skin tone mismatch from different lighting conditions.

Modular pipeline architecture

Rebuilt from a single 500-line demo.py into 6 focused modules. Each file has one responsibility, making debugging fast.
